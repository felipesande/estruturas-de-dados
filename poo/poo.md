# Programação Orientada a Objetos (POO)

## 1. Definição
A **Programação Orientada a Objetos (POO)** é um paradigma de programação baseado na organização do código em torno de **objetos**, que são instâncias de **classes**. 

> A POO permite criar softwares mais modulares, reutilizáveis e fáceis de manter.

---

## 2. Os 4 Pilares da POO
As principais características (ou pilares) que sustentam esse paradigma são:

* **Encapsulamento:** Processo de ocultar os detalhes internos de um objeto (dados), fornecendo uma interface pública segura para interagir com ele.
* **Herança:** Permite que uma classe herde propriedades e comportamentos (métodos) de outra classe, evitando repetição de código.
* **Polimorfismo:** Permite que objetos de diferentes classes sejam tratados como objetos de uma mesma classe comum, executando comportamentos específicos.
* **Abstração:** Processo de identificar as características essenciais de um objeto, ignorando detalhes complexos e irrelevantes.

---

## 3. Vantagens da POO
O uso deste paradigma oferece benefícios claros no dia a dia do desenvolvedor:

1.  **Maior Modularidade:** O sistema é dividido em partes menores e independentes.
2.  **Reutilização de Código:** Classes podem ser reaproveitadas em diferentes partes do projeto ou em outros sistemas.
3.  **Facilidade de Manutenção:** Corrigir erros ou adicionar funções torna-se mais simples devido à organização.
4.  **Modelagem Realista:** Facilita a representação de entidades do mundo real (ex: Cliente, Conta Bancária, Produto) dentro do código.

---

## 4. Resumo
A POO é um paradigma poderoso que torna o desenvolvimento mais eficiente e eficaz. Dominar seus pilares é o primeiro passo para construir arquiteturas de software profissionais e escaláveis.
